# COSMO-гейт: σ₈-експеримент на Apple M3 Max (48 GB) — повна інструкція

**Що це.** Останній незакритий гейт Paper 5: перевірка центрального передбачення статті
(лінійний score-сурогат структурно недобачає **амплітудні/масштабні** параметри; парна
PATP-фіча це лагодить) на **космологічному weak-lensing бенчмарку самих авторів FSM** —
LSST Y10 lognormal-карти конвергенції, 6-вимірне зведення від навченого ResNet18-VMIM
компресора, параметр інтересу **σ₈** (компонент №2, амплітуда спектра потужності).
Увесь конвеєр — camera-ready код авторів (vendored у цьому репо); наша зміна — **одна
фічемапа** (`fit_feature_sm` = їхній `fit_linear_sm` з одним заміненим рядком).

**Чому M3 Max достатньо.** GPU тут не обов'язковий: увесь стек — чистий JAX, який
працює на CPU. Пам'ять не є вузьким місцем (пік — кілька ГБ на vmap-батч 100 карт
256×256×5; 48 GB вистачає з великим запасом). Вузьке місце — **час**: тому протокол
скорочено (парний дизайн зі спільними сідами замість 100 прогонів авторів) і вбудовано
чекпойнти — прогін можна переривати й продовжувати.

## 1. Передумови на машині

- macOS на Apple Silicon (M3 Max), ~10 GB вільного диска.
- Python 3.11 або 3.12 **arm64** (перевірити: `python3 -c "import platform; print(platform.machine())"` → `arm64`).
- Xcode Command Line Tools (`xcode-select --install`), `git`, за потреби `git-lfs`
  (`brew install git-lfs && git lfs install`) — файли компресора можуть жити в LFS.

## 4. Середовище

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -U pip && pip install -r requirements-cosmo.txt
python check_env.py
```

`check_env.py` мусить надрукувати `== READY ==` і **оцінку годин** для планів A/A+/B.

**Tier 2 (fallback), якщо READY не досягнуто** через несумісність haiku/tfp/jax_cosmo
із свіжим jax — постав пінований набір у чистий venv:

```bash
pip install "jax==0.4.35" "jaxlib==0.4.35" "dm-haiku==0.0.13" "optax==0.2.4" \
            "tensorflow-probability==0.25.0" "jax-cosmo==0.1.0" scikit-learn numpy
```

**Важливо:** фінальні прогони — **тільки CPU-бекенд** (`jax.default_backend()=="cpu"`).
`jax-metal` не ставити / деінсталювати: він експериментальний, частина опів
(eigh/FFT/lax.scan-патерни цього симулятора) непідтримана або недетермінована —
результати не будуть відтворювані.

## 5. Smoke-тест (~5–10 хв)

```bash
python cosmo_gate_experiment.py --smoke
```

Друкує похибку одного вкороченого прогону + **екстраполяцію хвилин на повний прогін**
і годин на арм. Якщо smoke падає — розділ «Troubleshooting» нижче.

## 6. Основний експеримент

**План A (базовий, обов'язковий): 2 арми × 20 прогонів, n_obs=250.**

```bash
caffeinate -is python cosmo_gate_experiment.py --arm linear --runs 20 2>&1 | tee -a linear.log
caffeinate -is python cosmo_gate_experiment.py --arm poly2  --runs 20 2>&1 | tee -a poly2.log
```

**План A+ (якщо оцінка з check_env ≤ ~36 год):** додати третій арм

```bash
caffeinate -is python cosmo_gate_experiment.py --arm patp05 --runs 20 2>&1 | tee -a patp05.log
```

**План B (якщо оцінка > ~60 год):** `--runs 10` для обох армів (мінімум для парного
t-тесту — 8 пар; краще не опускатись нижче 10).

Практичні нотатки:
- `caffeinate -is` не дає машині заснути; `tee -a *.log` зберігає таймінги.
- Прогони **чекпойнтяться поштучно** (`results/{arm}_runNNN.pkl/.json`): Ctrl-C або
  перезавантаження безпечні, повторна команда продовжить із місця зупинки.
- Арми можна ганяти послідовно або в двох терміналах паралельно (кожен однопроцесний;
  XLA сам використає всі performance-ядра — паралель двох армів дасть ~1.5×, не 2×).
- Нічого в гіперпараметрах не міняти: N_ITER=100, N_PROP=25, N_SIM_DST=4, σ=1e-3,
  lr=1e-2, λ=1.0, Polyak-хвіст 50 — це **бюджетна чесність** порівняння (усі арми
  бачать однакову кількість симуляцій та однакові сіди `PRNGKey(2026000+r)`).

## 7. Аналіз і вердикт

```bash
python analyze_results.py
```

Друкує покомпонентну таблицю похибок (натуральний простір) і **pre-registered** гейт:

- **COSMO-1 (напрям):** середнє парне Δ = |σ₈-похибка(linear)| − |σ₈-похибка(poly2)| > 0.
- **COSMO-2 (сила):** однобічний парний t ≥ **1.70**.
- **[info, без гейта]:** patp05/patp07 проти poly2, покомпонентна таблиця, агрегатний MSE.

Пише `results/cosmo_gate_verdict.json`. **Критерії зафіксовані до прогону — не
міняти після побачених даних.** Чесний FAIL — теж публікований результат (межа
застосовності: компресоване зведення може виявитись достатньо гаусовим — тоді це
підтверджує *першопорядковість* гаусового стиснення на цих картах, і в статті це
чесно описується).

## 9. Troubleshooting

| Симптом | Причина/лік |
|---|---|
| `Couldn't locate the JAX substrate inside tensorflow_probability` | стара/нова tfp: постав Tier 2 (`tensorflow-probability==0.25.0`) |
| Помилки в `haiku._src.nets.resnet` / `ResNet18(...)` | несумісний dm-haiku: Tier 2 (`dm-haiku==0.0.13`, `jax==0.4.35`) |
| `pickle.load` падає на файлах компресора | файли — LFS-заглушки (розмір ~130 байт): `git lfs install` і перекачати |
| Дуже повільно (>40 хв/прогін у smoke-екстраполяції) | перевір `jax.default_backend()=='cpu'` і що немає Rosetta-python (arm64!); закрий важкі фонові процеси |
| `jax.errors...` про непідтримані опи на METAL | видалити jax-metal: `pip uninstall jax-metal` |
| NaN/розбіжність в одному прогоні | зафіксувати номер прогону і продовжити (у вердикті пара з NaN виключиться); повідомити — розберемо окремо |
| Перервали посеред прогону | нічого не робити: незавершений прогін не записаний, повторна команда переганяє лише його |

## 10. Що заборонено міняти (чесність порівняння)

| Ручка | Значення | Чому фіксовано |
|---|---|---|
| N_ITER/N_PROP/N_SIM_DST | 100/25/4 | бюджет авторів; однаковий для всіх армів |
| σ (proposal), lr, λ | 1e-3 / 1e-2 / 1.0 | значення авторів |
| Сіди | `PRNGKey(2026000+r)`, спільні між армами | парний дизайн |
| θ_init | `sample_lensing_prior(PRNGKey(0))` | як у скрипті авторів |
| Пороги COSMO-1/2 | Δ>0, t≥1.70 | pre-registered до даних |
| Симулятор/компресор | без змін | інакше це інший бенчмарк |

Дозволено: `--runs` (план A/B), порядок армів, паралель армів у двох терміналах,
`--n-obs` лише якщо погоджено окремо (тоді для ВСІХ армів однаково).
